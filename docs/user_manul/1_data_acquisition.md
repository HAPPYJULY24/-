## 1. 数据抓取模块 (Data Fetcher)

数据抓取界面是整个量化流水线的起点，负责从 Yahoo Finance、TradingView 及各加密货币交易所获取实时与历史 OHLCV 行情数据。

![数据获取界面截图](file:///D:/personal/quant/Quant/-/docs/user_manul/images/data_acquisition_ui.png)
> *(注：操作时请对照主窗口第一标签页 `📥 数据抓取`)*

### 1.1 资产类型与代码校验规范

在 **输入配置 (Input Configuration)** 区域，系统提供了 5 种资产类型。由于底层的接口适配器（YFinance、TradingView、CCXT）对输入代码的格式有严格要求，UI 设有**防御性正则校验器**，并在后台执行**自适应名称转换**：

| UI 资产类型 | 匹配适配器 | 验证器规则 | 典型输入示例 | 自动前处理逻辑 (Facade) | 结算货币 | 原生交易单位 |
| :--- | :--- | :--- | :--- | :--- | :---: | :---: |
| **马股** | `YFinance` | 最多4位纯数字 | `1155` (Maybank) | 自动追加 `.KL` 后缀，转换为 `1155.KL` | `MYR` | `share` (股) |
| **美股** | `YFinance` | 字母与小数点 (最多10位) | `AAPL` / `BRK.B` | 自动转换为纯大写 | `USD` | `share` (股) |
| **国际期货 (YF)** | `YFinance` | 允许字母、数字、等于号及特殊符号 | `GC=F` (黄金) / `CL=F` | 若无 `=F` 或 `=X` 后缀，系统自动追加 `=F` | `USD` | `contract` (手) |
| **Bursa期货 (TV)** | `TradingView` | 字母、数字及感叹号 | `FCPO1!` (棕榈油) / `FKLI1!` | 1. 自动判定是否为 CBOT 农业代理合约（如 `ZL` 豆油, `ZS` 大豆）；<br>2. 真实马交所期货自动补全 `MYX:` 前缀 | `MYR` (真实)<br>`USD` (CBOT) | `contract` (手)<br>`lb`/`bushel` |
| **加密货币** | `CCXT` | 允许字母、数字及斜杠 (最多20位) | `BTC/USDT` | **显示隐藏控件：“交易所”** (Luno, Binance, OKX, Bybit)。解析 `/` 后半部分作为结算币种。 | `USDT`等 | `coin` (个) |

### 1.2 时间粒度与防御性历史限制

为了防止由于 API 溢出或平台超出数据限制而导致请求被拒，UI 控件在您选择 **时间粒度 (Timeframe)** 和 **日期范围 (Date Range)** 时，会自动激活 **安全拦截逻辑**。若不符合以下硬性规定，系统将置灰 `获取数据` 按钮并给出红色高亮警告：

> [!WARNING]
> #### ⏰ 粒度与历史区间获取上限限制：
> 1. **`1m` (1分钟线)**：Yahoo Finance API **仅支持获取最近 7 天内** 的历史数据。若 `开始日期` 距离今天超过 7 天，系统强制拦截。
> 2. **`5m / 15m / 1h` (日内K线)**：Yahoo Finance **仅支持获取最近 60 天内** 的历史数据。若超过 60 天，请求将被拦截。
> 3. **`1d` (日线及以上)**：无硬性上限限制，但有最小时间跨度约束（如选择 `1y` 粒度，日期跨度必须 $\ge 365$ 天）。

### 1.3 高级网络与时段配置

为应对复杂的网络环境及个性化的交易回测需求，界面下方提供了两个折叠面板：

#### 🌐 网络设置 (Network Settings)
* **启用代理 (Enable Proxy)**：勾选后，可输入代理 URL。
* 默认值：`http://127.0.0.1:7890`（支持主流本地网络代理端口）。由于国内访问 Yahoo Finance 接口极易遭遇网络波动，**强烈建议在下载美股/国际期货时开启此代理**。

#### ⚙️ 高级设置 (Advanced Settings) - v2.0
* **✨ 启用增量更新 (Incremental Update)**：
  * **机制**：开启后，系统在下载前首先检索本地 `Master DB`。若已有该资产的历史数据，将自动提取本地最新时间戳作为起始点，仅向 API 增量下载最新行情并合并。
  * **优势**：**可节省高达 80% 的下载时间和网络流量**，完美避开重复请求被封 IP 的风险。
* **应用交易时段过滤 (`apply_session_filter`)**：
  * **马股 & Bursa期货**：由于交易所存在固定的午休，系统将**强制开启过滤且不允许手动修改**。
    * **马股午休过滤**：剔除 `12:30 ~ 14:30` 之间的无成交K线。
    * **Bursa期货午休过滤**：剔除 `12:45 ~ 14:30` 之间的无成交K线。
  * **美股、外盘期货、加密货币**：默认关闭（视为 24 小时或连续交易）。勾选此项可激活 **自定义交易时段输入框**，支持设定个性化活跃区间（如加密货币只保留 `20:00` 至次日 `04:00` 的深夜波动活跃段），系统支持跨日/夜盘时段判定。

### 1.4 底层自动化处理机制（汇率/防崩溃/去重）

在您点击 `获取数据 (Fetch Data)` 后，底层适配器链（Facade 模式）会自动在后台线程进行多维度的数据清洗，这一机制对 UI 保持流畅至关重要：

> [!NOTE]
> 1. **隐式汇率本币对齐 (USD ➔ MYR)**：
>    * **痛点**：量化回测必须在统一的本币环境下进行（本系统默认为林吉特 `RM / MYR`），但美股和黄金等资产以 `USD` 结算。
>    * **自动转换**：若系统检测到下载资产为 `USD` 结算，会**自动在后台静默发起**对 `USDMYR=X`（美元兑林吉特）汇率数据的联动抓取（日内线自动拉取 1h 汇率，日线拉取 1d 汇率），并利用底层插值算法，将资产的 Open/High/Low/Close 实时换算为 `MYR`。
> 2. **防崩溃价格钳位 (Price Clamping)**：
>    * 系统自动扫描 OHLCV，如发现负数价格或零价格（常因 API 数据源脏数据引起），会将其自动替换为 `NaN` 并执行**前向填充 (ffill)**。若整段数据缺失，则强制钳位至最小 Tick `1e-5`，**彻底杜绝 Numba 核心计算引擎中因除以零导致的底层闪退/崩溃**。
> 3. **极值与空值清洗 (Data Sweeper)**：
>    * **去重**：合并新旧数据时，系统根据时间戳进行去重，当时间戳冲突时**保留最新抓取的值 (`keep='last'`)**。
>    * **剔除非交易日**：对于日线数据，系统自动剔除 `Volume = 0` 的非交易日（如周末和法定节假日）。*(注意：由于汇率数据本身没有成交量，外汇 `FX` 类别不受此限制)*。

### 1.5 数据管理与导出

抓取完成后，数据预览网格 (Data Grid) 将高亮显示前 5 行与后 5 行，同时激活数据操作按钮：
* **导出 CSV / 导出 Parquet**：将当前预览的干净数据保存至本地电脑任意位置。
* **💾 保存至数据中心 (Save to Master DB)**：将数据以标准规范写入 `datacenter/RawData` 对应子目录（如 `MY_stock/`, `US_stock/`, `BF/`, `IF/`, `Crypto/`）。
  * 写入时，系统自动对文件名中的非法字符进行安全转义（例如：`BTC/USDT` 转存为 `BTC-USDT_1d.parquet`；`MYX:FCPO1!` 转存为 `MYX-FCPO1!_15m.parquet`），确保在 Windows 操作系统下完美兼容。

## 2. 数据对齐工作室 (Data Alignment Studio)

在多资产交叉盘交易、套利交易（例如：马盘棕榈油 `FCPO` 关联芝商所豆油 `ZL`）或多因子挖掘中，不同市场的开市时间、节假日以及时区存在天然差异。**数据对齐工作室**就是解决这一痛点的终极武器。

![数据对齐界面截图](file:///D:/personal/quant/Quant/-/docs/user_manul/images/data_alignment_ui.png)
> *(注：点击主窗口菜单栏 `工具 (Tools)` ➔ `数据对齐工作室 (Data Alignment)` 即可唤出)*

### 2.1 基准资产 (Anchor Asset) 对齐原理

数据对齐工作室的核心设计思想是 **基准资产 (Anchor Asset)** 驱动机制：

```mermaid
graph TD
    A[选择基准资产 Anchor: 如 FCPO] --> B[生成绝对时间轴 Timeline]
    C[参考资产 A: ZL] --> D[按照 Timeline 匹配时间戳]
    E[参考资产 B: USDMYR=X] --> D
    D --> F{是否缺失该时刻数据?}
    F -- 是 --> G[根据前向填充 ffill 补齐上一个有效价格]
    F -- 否 --> H[保留原始值]
    G --> I[生成规整矩阵: datetime | Anchor | Ref_A_ffill | Ref_B_ffill]
    H --> I
```

1. **绝对时间主宰**：以选定的基准资产（标有 ⚓ 符号）的交易时间戳为**绝对基准**。
2. **时序规整**：参考资产在该时刻若未交易，自动以前一次的有效价格进行**前向填充 (Forward Fill)**，确保因子矩阵在任何时间点均无空值。

---

### 2.2 多文件对齐参数详解

数据对齐工作室支持 **2 至 5 个数据源** 的灵活对齐：

1. **文件选择 (`alignment.select_files`)**：
   * **Asset A & Asset B (必选)**：对齐的两个核心数据源。
   * **`alignment.multi_align_checkbox` (启用多数据对齐)**：勾选后，下方将展开 Asset C, Asset D, Asset E 三个可选下拉框，允许一次性对齐 5 个资产。
   * **⚙️ 互斥选择机制 (Mutual Exclusion)**：为了防止手误选重，**已被任意下拉框占用的文件，在其他下拉框中会自动变灰并被禁用**。
2. **基准选择 (`alignment.anchor_label`)**：
   * 对应单选按钮组（Asset A 至 E）。勾选哪个，哪一项就成为时间轴的 Anchor（UI 标签上会同步显化 ⚓ 标志）。
3. **前向填充 (`alignment.forward_fill`)**：
   * 默认勾选。不勾选时，未对齐时间点将保持为 `NaN`（在下方预览表格中会以**粉红色底色**醒目高亮提示）。
4. **仅保留重叠时间段 (纯净模式)**：
   * 默认关闭。如果勾选，系统会在对齐后自动扫描 `is_overlap` 辅助列，**彻底剔除因单边交易或节假日单边开市造成的非重叠行**，仅保留所有标的都在交易的“黄金重叠时间段”，这是高频套利策略的强力配置。

### 2.3 异常拦截与防呆机制

为了保证数据库的一致性，防止“套娃对齐”导致的数据混乱，工作室内置了强大的防呆审查：

> [!IMPORTANT]
> #### 🛑 对齐文件防呆拦截：
> 1. **严禁二次对齐**：系统在加载文件时，会读取 Parquet 文件的 Metadata。如果检测到待选文件已经是经过对齐处理的文件（即包含 `is_overlap` 或已对齐特征列），对齐线程将**强行中断**，并弹出警告：`“该文件已经是经过对齐处理的数据文件，无法作为源数据再次对齐！”`。
> 2. **完全去重校验**：在开始对齐前，系统自动扫描所有激活的 Combos，若检测到不同下拉框内有重复文件路径，立刻阻断执行，防止逻辑自循环崩溃。

---

### 2.4 对齐数据的保存与管理

* **刷新列表 (`alignment.buttons.refresh`)**：手动重新扫描 `datacenter/RawData` 中的最新文件。
* **预览数据 (`alignment.preview.title`)**：对齐完成后，表格实时展示对齐数据前 100 行。`NaN` 值用红底表示，让缺失值分布一目了然。
* **导出结果 (`alignment.buttons.export`)**：支持将完整对齐数据保存为 `.parquet` 或 `.csv` 格式至桌面。
* **💾 保存至数据中心**：
  * 点击后弹出输入框，输入数据集名称（默认前缀为 `Aligned_`，如 `Aligned_FCPO_ZL1_15m`）。
  * 系统自动过滤文件名中的非法特殊字符，确保仅保留英文字母、数字、下划线及横杠。
  * **一键聚拢归档**：文件将自动安全归档至数据中心的专用对齐库中：
    `D:\personal\quant\Quant\-\datacenter\RawData\alignment\<您的命名>.parquet`
  * 该对齐数据集将作为标准多资产行情源，供后续 **因子挖掘 (Alpha Research)** 模块直接调用。
